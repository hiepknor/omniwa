# Release Checklist

## Candidate Readiness

- [ ] Scope is traceable to approved docs.
- [ ] Required tests passed.
- [ ] Architecture boundary checks passed.
- [ ] Security/redaction checks passed.
- [ ] Persistence migration or data impact is reviewed, if any.
- [ ] Backup/restore impact is known.
- [ ] Rollback plan is documented.
- [ ] Operational smoke checks are defined.

## Hotfix Readiness

- [ ] Incident or defect is documented.
- [ ] Minimal patch scope is defined.
- [ ] Regression test exists or gap is documented.
- [ ] Security and data retention rules are preserved.
- [ ] Rollback or forward-fix path exists.

## Post Release

- [ ] Smoke checks completed.
- [ ] Monitoring reviewed.
- [ ] Known issues documented.
- [ ] Follow-up issues created.
