# OmniWA Event Contracts

## Purpose

This document defines business-level contracts for OmniWA events.

It does not define payload JSON, schemas, database tables, event bus topics, queues, Kafka, BullMQ, REST APIs, Prisma models, serializers, or source code.

## Contract Baseline

Every event contract includes these conceptual fields:

| Contract Item | Meaning |
| --- | --- |
| Event Name | Stable product event name from `EVENT_CATALOG.md`. |
| Event Version | Contract version such as v1 or v2. |
| Event Identity | Opaque event identity for idempotency and traceability. |
| Occurred At | Business occurrence time or translated observation time. |
| Producer | Aggregate, Application, or Infrastructure boundary that produced the event category. |
| Correlation Context | CorrelationId and safe causation reference where available. |
| Data Classification | Public/Internal/Confidential/Secret handling marker. Secret values are never included. |

Required Data below means data required in addition to the baseline. Optional Data means safe data that may be included when available and approved.

## Application And Infrastructure Contract Rules

Application and Infrastructure Events follow the same baseline but are not Domain Events:

| Event Family | Purpose | Producer | Consumer | Required Data | Ordering | Idempotency | Version Strategy |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Application work-request events | Request workflow work such as outbound send, media processing, webhook delivery, reconnect, retention cleanup, health refresh, or audit write. | Application. | WorkerJob or owning Application workflow. | Work type, owner context reference, correlation context, idempotency key. | After source domain fact and precondition checks. | Idempotency key required. | v1 per work category; additive changes only. |
| Infrastructure provider observation events | Bring provider-native observations to Application translation boundary. | Provider adapter. | Application. | Provider profile reference, observation category, safe external reference. | Provider order is best effort; Application must classify stale observations. | External reference and observation identity dedupe. | Adapter-private versioning; not external contract. |
| Infrastructure transport/queue/dependency events | Report transport, queue, worker, or dependency observation. | Infrastructure adapter. | Application. | Boundary, outcome category, safe reason, correlation context. | Best effort; owner aggregate decides state changes. | Boundary observation identity dedupe. | Adapter-private versioning. |
| Integration events | Deliver sanitized product facts externally. | Webhook Delivery. | External webhook consumers. | Source signal reference, event version, safe product metadata, idempotency key. | Per source aggregate where documented, otherwise eventual. | External delivery idempotency key required. | Public versioning rules in `EVENT_VERSIONING.md`. |

## Contract Rejection Rules

An event contract must be rejected if:

- It includes Secret values.
- It requires raw Confidential payloads by default.
- It depends on provider-native payload names.
- It embeds database, queue, API, transport, or implementation naming.
- It allows a non-owner context to mutate source aggregate state.
- It introduces campaign, broadcast, group administration, or unsupported message type commitments.
